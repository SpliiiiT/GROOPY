Part 1

How the code works — the 4 core ideas

1.1 Architecture at a glance

                 ┌───────────────────────────────────────────┐
                 │  shared/  (single source of truth)        │
                 │  contract.py · vocabulary.py · config.py  │
                 └──────────────┬─────────────┬──────────────┘
     Sign → Text                │             │            Text → Sign
 ┌──────────────────────────────┘             └──────────────────────────────┐
 │ TRACK A — recognition/                       TRACK B — synthesis/          │
 │ camera → preprocess/holistic → CNN/seq       text/speech → gloss → SignPlan│
 │ → TokenStream (gate/debounce) → Token        → player (clips / letters)    │
 └────────────────────────────────────────────────────────────────────────────┘

The Token (shared/contract.py) is the one object that flows between tracks. Recognition emits Tokens; Synthesis consumes the same shape. Define it once → both directions agree by construction.

1.2 The live recognition pipeline — the four guards

The raw model is never trusted directly. Four guards sit between a prediction and a committed character. This table is the whole live story — memorize where each one lives:

GuardPurposeWhere in code
Hand-presence gatedon't predict with no hand on screenpreprocess.crop_hand → None; word_stream hand_activity<0.3
Confidence gateemit only if p ≥ 0.80token_stream.update
Debounce≤ 1 emit per 500 mstoken_stream.update
No-repeat windowsame label can't re-fire for 2× windowtoken_stream.update
Capture-to-commitletter added only on a keypressdesktop app calls .update() on demand
The key architectural decision

The model produces a guess every frame, but TokenStream is a stateful filter that turns a noisy stream of guesses into a few clean, committed Tokens. That's what stops the demo spewing 30 letters/second.

1.3 The shared "contract" & 1.4 Reproducibility

  • Contract: contract.py (the Token/Sentiment shapes) + vocabulary.py (the 20 words that are both the word-model classes and the clip keys). Change the list once, both tracks update; anything off-list is fingerspelled.
  • Reproducibility: SEED = 42 seeds every split, shuffle, and run, so the bake-off is fair. paths.app_root() makes paths repo-relative → the same code runs locally, on Colab, and as a .exe.

2.1 shared/contract.py

The data contract — the single object that flows between tracks.

@dataclass
class Token:
    token: str            # "a" (letter) | "hello" (gloss) | "space" (control)
    confidence: float     # 0..1, only emitted at/above the gate
    timestamp: int        # epoch ms, stamped at prediction time
    kind: str             # "letter" | "word" | "control"
    sentiment: Optional[Sentiment] = field(default=None)   # v2, backward compatible
  • token is the normalized payload; kind tells Track B whether to play a word clip, a letter, or act on a control.
  • sentiment defaults to None → a v1 consumer that ignores it still works (additive change).
  • to_dict/from_dict are the JSON serialization seam; from_dict(None) returns None so tokens without sentiment round-trip cleanly.
  • CONTROL_TOKENS = {"del","nothing","space"} — note "del", matching the dataset folder names exactly.

2.3 recognition/src/preprocess.py — static hand cropping

Crop to the hand before classifying, so the CNN learns the hand shape, not the background. Same function offline & live → no skew.

def crop_hand(bgr_image, margin=0.25, static=True):
    results = _get_hands(static).process(rgb)
    if not results.multi_hand_landmarks:
        return None                      # ← the hand-presence gate
    ... bounding box from landmarks, expand 25% each side, clamp to frame ...
    return bgr_image[y1:y2, x1:x2]

def preprocess_for_model(bgr_image):
    resized = cv2.resize(rgb, (224,224), interpolation=cv2.INTER_AREA)
    return resized.astype(np.float32) / 255.0    # [0,1]; backbones rescale internally
  • MediaPipe is a cached singleton, imported lazily so the module still loads without it.
  • crop_hand returns None when no hand — the static-path hand gate. The box is expanded 25% so fingertips aren't clipped.
  • Output is [0,1]; each backbone applies its own rescaling, so all candidates get identical input.
  • crop_and_preprocess falls back to the full frame if no hand, so live playback never goes blank (the confidence gate suppresses the weak guess anyway).

2.4 recognition/src/data.py — the CNN dataset

_augment_layer(): RandomRotation(.08), RandomZoom(.10), RandomTranslation(.08,.08),
                  RandomBrightness(.15), RandomContrast(.15)   # NO horizontal flip

train_ds = image_dataset_from_directory(subset="training",  seed=SEED, ...)
holdout  = image_dataset_from_directory(subset="validation", seed=SEED, ...)
val_ds = holdout.take(val_batches); test_ds = holdout.skip(val_batches)   # test peeled off val
train_ds = train_ds.map(_normalise).shuffle(32, seed=SEED).map(augment).prefetch(AUTOTUNE)
  • No horizontal flip — flipping can turn a valid ASL handshape into an invalid one. Brightness/contrast help generalize across skin tones (fairness, not just accuracy).
  • Test images are peeled off the validation stream (take/skip) so they're never seen in training; all splits deterministic given SEED.
  • Two Colab OOM lessons are baked in: cache=False (87k imgs ≈ 40 GB) and shuffle(32) not 1000 — the dataset is already batched, so 1000 buffers ~19 GB and crashes.

2.5 recognition/src/models/ — the CNN candidates

  • cnn_scratch: _conv_block = [Conv2D(3×3, use_bias=False) → BN → ReLU]×2 → MaxPool at 32/64/128/256. use_bias=False because the following BatchNorm cancels the bias. The 256-block is the Grad-CAM target.
  • Backbones: wrapped with input_tensor=x so their layers are inlined into the outer graph (this is what lets Grad-CAM reach them). Each stores base_layer_names (names, not the object — storing the object would double-track layers and corrupt .keras save/load).

2.6 recognition/src/train.py — two-phase fine-tuning

_compile(model, LEARNING_RATE)                       # Phase 1: LR 1e-3, head only
model.fit(train_ds, val_ds, callbacks=[EarlyStopping(restore_best_weights=True),
                                       ReduceLROnPlateau(factor=0.3, patience=2)])
if is_pretrained:
    for l in backbone[:-30]: l.trainable = False     # unfreeze only last ~30
    for l in backbone:
        if isinstance(l, BatchNormalization): l.trainable = False   # keep BN frozen
    _compile(model, LEARNING_RATE/10.0)              # Phase 2: LR 1e-4
    model.fit(...)
  • Phase 1 trains the head; Phase 2 (pretrained only) unfreezes the last ~30 layers, keeps BatchNorm frozen, recompiles at LR/10. cnn_scratch skips Phase 2.
  • EarlyStopping(restore_best_weights=True) rolls back to the best epoch — a core overfitting guard.

2.7 recognition/src/holistic.py — landmarks + the decisive fix

One BGR frame → a 258-feature vector (pose + both hands). Same offline & live. The most important file in the word track.

landmarks = [pose 33*(x,y,z,vis) | LH 21*(x,y,z) | RH 21*(x,y,z)]  # = 258, missing→zero

def normalize_sequence(seq):                     # ← THE fix: 5% → 37% → 70%
    cx,cy  = shoulder midpoint (pose 11 & 12)    # translation reference
    scale  = shoulder width = hypot(Lx-Rx, Ly-Ry) # scale reference
    nx = (x - cx) / scale;  ny = (y - cy) / scale # recenter + rescale x,y only
    # z, visibility untouched; zero/absent landmarks & pose-less frames left as-is
  • Missing pose/hands are zero-filled so the vector is always 258 — the model's input shape must be fixed.
  • Why it matters: without it the model confuses where the signer stands with what they sign. Recentering on the shoulders + dividing by shoulder width makes features translation- and scale-invariant → hand motion relative to the body. This one change took test accuracy 5% → 37% → 70%.
  • Only x,y are normalized (z/visibility are noisy). The present/valid masks keep padding as padding.
  • video_to_sequence turns a training clip into (30, 258) by uniformly sampling 30 frames (honoring WLASL sub-ranges).

2.8 recognition/src/sequence_data.py — the word dataset

_pad_or_truncate(seq, 30)   # exactly (30,258): truncate if longer, zero-pad if shorter

def augment_sequence(seq, rng):
    out = where(present, out*scale + shift, 0)      # signer distance/position
    out = where(present, out + normal(0,.012), 0)   # light landmark jitter
    out = out[speed_warped_indices]                 # signing speed 0.8–1.2×

seq = normalize_sequence(load(npy))   # SAME normalization as live → no skew
augment TRAIN only (×8), val/test stay clean
  • Three physically-meaningful augmentations, each masked with present so zero-padding is never perturbed into fake data. No mirroring — it would change some signs.
  • Applies the exact normalize_sequence used live (anti-skew), splits deterministically, then augments the training set only — essential given ~40 samples/class.

2.9 recognition/src/word_models.py — 4 sequence candidates

  • All start Input(30,258) → Masking(0.0). Masking tells the RNN/attention to skip zero-padded frames, so a short sign isn't diluted by empty timesteps.
  • The transformer adds a learned Embedding(30, d_model) positional encoding — self-attention is order-invariant, so it must be told which frame is which. (Full layer tables in Part 4.)

2.10 recognition/src/train_word.py

Mirror of train.py but single-phase (these train from scratch): same callbacks, Adam 1e-3, augmentation ×8. Saves word_{name}.keras + records test_accuracy/params for the scorer.

2.11 recognition/src/token_stream.py — the live "brain"

Feed it (label, confidence); it returns a Token or None. Where the four guards actually execute.

def update(self, label, confidence, sentiment=None):
    if confidence < self.gate: return None                            # ① confidence gate
    if now_ms - self._last_emit_ms < self.debounce_ms: return None    # ② debounce
    if label == self._last_token and (now_ms-self._last_emit_ms) < debounce_ms*2:
        return None                                                   # ③ no-repeat window
    self._last_token = label; self._last_emit_ms = now_ms
    return Token(token=..., confidence=round(c,3), timestamp=int(now_ms), kind=..., sentiment=...)
  • Stateful: remembers the last label and when it fired. Both the CNN path and WordStream own one.
  • ① below 0.80 → nothing. ② within 500 ms → nothing (caps emit rate). ③ the same label can't re-fire within 2× the window, so holding a letter steady yields one token, not a stream.
  • Control labels are promoted to kind="control"; letters/words are lowercased.

2.12 recognition/src/word_stream.py — live word recognition

self._buf = deque(maxlen=30)          # rolling window: push a frame, oldest drops off

def push(self, landmark_vec):
    self._buf.append(vec)
    if not self.ready: return None                                     # wait for 30 frames
    hand_activity = mean(any(stacked[:, -126:] != 0, axis=1))          # both hands present?
    if hand_activity < 0.3: return None                               # ← word hand-presence gate
    seq = normalize_sequence(stacked)                                  # same as training
    probs = self.model.predict([seq])[0]
    return self._stream.update(INDEX_TO_GLOSS[argmax], float(max))     # same gate/debounce
  • A deque(maxlen=30) is the sliding window; it predicts every frame once full.
  • Clever gate: the model has no "not signing" class, so it always argmaxes some word. Looking at the last 126 features (both hands) and suppressing when hands are essentially absent stops idle noise surfacing a spurious word.
  • Normalizes identically to training, then routes through the same TokenStream — words get the same treatment as letters.

2.13–2.14  xai_gradcam.py · evaluate*.py

  • Grad-CAM (Part 6): _last_conv_layer_name descends into nested backbones and handles the Keras-2→3 API change; make_gradcam_heatmap uses tf.GradientTape on a model that outputs (last-conv activations, predictions).
  • Evaluators (Part 5): loop every *.keras, compute accuracy + macro-F1 + confusion matrix + latency (via direct model(x), not .predict — dispatch overhead would inflate a metric worth 20%) + size, then feed scorecard.score. Metrics are pinned to all 29 labels so a small test split can't crash sklearn. Robustness is a manual dict from the Grad-CAM review; unreviewed models are printed, never faked.

Synthesis, file by file

The flow: synthesize() → text_to_gloss → build_sign_plan → apply_sentiment → play_sign_plan.

3.1 synthesis/src/text_to_gloss.py

_STOPWORDS = {a, an, the, is, are, am, to, of, do, does}   # ASL usually omits these
_SYNONYMS  = {hi→hello, hey→hello, thank→thanks, wanna→want, ...}

def normalise(text):   # NFKD strip accents → casefold → strip punct → collapse spaces
def text_to_gloss(text, drop_stopwords=True):
    for w in normalise(text).split():
        if drop_stopwords and w in _STOPWORDS: continue
        glosses.append(_SYNONYMS.get(w, w))    # synonym-map, else keep word
  • Deliberately simple, deterministic, dependency-free — a PoC rule set, not real ASL grammar.
  • Stopword list is small on purpose (over-dropping hurts meaning). Synonyms steer common words toward glosses that have clips ("hi" → the hello clip).
  • Words with no clip survive here and get fingerspelled downstream — never a dead end.

3.2 synthesis/src/gloss_to_signplan.py

@dataclass WordClip:    gloss, clip_path;  hold_ms=0, repeat=1   # A2 defaults = no effect
@dataclass Fingerspell: word, letters[A-Z]; hold_ms=0, repeat=1

def build_sign_plan(glosses):
    for g in glosses:
        clip = vocab.resolve(g, clips_dir)          # known gloss? (the vocabulary lookup)
        if clip: steps.append(WordClip(g, clip))
        else:    steps.append(Fingerspell(g, _letters_of(g)))   # fallback: spell it
  • Pure/headless — builds the plan; player.py does I/O. Two step types; hold_ms/repeat default to "no effect" so an unmodified plan behaves as before.
  • vocab.resolve uses the same 20-word list the model classifies — this is where "known word vs fingerspell" is decided. Fingerspelling reuses the Recognition track's letter images (one asset set, both ways).

3.4 synthesis/src/pipeline.py — your Track B entry point

def apply_sentiment(plan, sentiment):     # Decision A2 — the sentiment seam
    if sentiment is None or sentiment.label == "neutral": return plan   # no-op
    if sentiment.score < 0.75: return plan                             # weak → no-op
    for step in plan.steps:
        if isinstance(step, WordClip):
            step.hold_ms = 400; step.repeat = 2      # held pause + replay, like tone of voice

def synthesize(text=None, wav=None, with_sentiment=True):
    if wav: text = transcribe(wav)               # speech & text converge here
    glosses = text_to_gloss(text)
    plan    = build_sign_plan(glosses)
    plan    = apply_sentiment(plan, _analyze(text))
    return Result(source_text=text, glosses=glosses, plan=plan, sentiment=...)
  • Only confident, non-neutral sentiment changes playback (held pause + replay on known-word signs). Neutral/weak/missing → a true no-op. Fingerspelled steps are left alone.
  • _analyze calls the sentiment module defensively — if it's missing or throws, synthesis still works; sentiment can never break the pipeline.
  • The whole track in one function: (optionally transcribe) → glosses → plan → annotate → Result.

3.5 synthesis/src/player.py — render the plan (OpenCV)

for step in plan.steps:
    if WordClip:
        if not clip_path.is_file(): report(); continue     # missing → skip, never crash
        for _ in range(step.repeat):                       # ← A2 replay
            play clip frame-by-frame at fps; q/Esc quits
        if step.hold_ms: waitKey(step.hold_ms)             # ← A2 held pause
    elif Fingerspell:
        show one image per letter, hold ~700ms each
  • The I/O layer — everything above it stays headless-testable. repeat/hold_ms (set by apply_sentiment) are honored here — that's how emphasis becomes visible motion.
  • Fully defensive: every missing or unreadable asset is reported and skipped, so a demo never hard-fails.

Part 6

XAI — explainability

Grad-CAM on the CNN, step by step

  • Take the last conv layer (256-block for cnn_scratch) — it still carries spatial resolution.
  • Build a grad_model outputting (those activations, the predictions).
  • With tf.GradientTape, get the gradient of the predicted class w.r.t. those activations.
  • Global-average the gradients → one weight per feature map.
  • Weighted sum → ReLU → normalize [0,1] → resize to 224² → overlay in jet.
What it revealed

EfficientNetB0 attends tightly to the hand (robust); cnn_scratch drifts to the background on hard cases. Hard pairs M/A, M/E, Q/G (predicted at EDA) are confirmed → closes the CRISP-DM loop. These feed the manual robustness score (EfficientNet 0.85, cnn_scratch 0.65). In sign language a model that cheats on background or skin tone is biased — so Grad-CAM is a trust/bias check, not decoration.

Other models: the word model is interpretable via landmark normalization (invariant geometry, not pixels), Masking, and its confusion matrix; the sentiment scratch model is intrinsically interpretable (per-word weights).
